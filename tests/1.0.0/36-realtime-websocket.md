# 36 实时 WebSocket 推送 — 冒烟测试

**前置条件**:已登录,≥1 Agent 在线。

| # | 测试场景 | 操作步骤 | 预期结果 | 阻断级 | 状态 |
|---|---------|---------|---------|--------|------|
| WS1 | 浏览器 WS 连接 | 登录后进入任意实时页 | `/api/ws/servers` 建立连接 | 是 | ✅ |
| WS2 | 指标实时推送 | 观察服务器详情 | 指标随 Agent 上报实时刷新(无需手动刷新) | 是 | ✅ |
| WS3 | 上线/离线事件 | Agent 上下线 | UI 即时反映 ServerOnline/Offline | 是 | — |
| WS4 | 能力变更推送 | 修改能力位 | 前端收到 CapabilitiesChanged 并更新 UI | 否 | — |
| WS5 | IP 变更推送 | Agent 外网 IP 变化 | 收到 ServerIpChanged 更新 | 否 | — |
| WS6 | 断线重连 | 中断网络后恢复 | WS 自动重连,数据恢复推送 | 是 | ✅ |
| WS7 | Docker 订阅 | 进入 Docker 页订阅日志 | 实时日志流推送,离开后取消订阅 | 否 | — |

> WS1 ✅: GET /api/ws/servers 无凭证返回 401(端点存在且强制鉴权);已登录浏览器会话下实时图表持续刷新,证明 WS 已建立。SPA 路由切换复用单一持久连接(架构良好)。
> WS2 ✅: 服务器详情 realtime 模式,6s 内图表像素差 0.33%、8s 内 0.16%,无手动刷新即随 Agent 上报刷新。
> WS6 ✅: 整页重载(=断开并重建 WS)后实时图表恢复推送(8s 像素差 0.16%),证明 WS 客户端自动重连。
> WS3 —: 共享测试 Agent 不可停止(子代理约束),无法制造上下线事件;"Online" 状态由 WS 驱动正确显示。
> WS4 —: B 组只读,严禁修改能力位,无法触发 CapabilitiesChanged。
> WS5 —: 无法强制改 Agent 外网 IP。
> WS7 —: Docker 能力位(128)默认关闭(caps=60),B 组只读,无法测试 Docker 日志流订阅。

**汇总**:✅ 3 / ❌ 0 / — 4(WS3/4/5/7 受共享只读环境约束未执行,非缺陷)
