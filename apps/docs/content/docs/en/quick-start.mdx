This guide walks you through the fastest path to a working ServerBee installation. Pick whichever method suits your environment.

## Prerequisites

You need one of the following on your server machine:

- **Docker and Docker Compose** (recommended for quick setup)
- A Linux (amd64/arm64) host with systemd, for the install script

## Option 1: Docker Compose (Recommended)

This is the fastest way to get started. Create a `docker-compose.yml` file:

```yaml
services:
  serverbee-server:
    image: ghcr.io/zingerlittlebee/serverbee-server:latest
    container_name: serverbee-server
    restart: unless-stopped
    ports:
      - "9527:9527"
    volumes:
      - serverbee-data:/data
    environment:
      - SERVERBEE_ADMIN__USERNAME=admin
      - SERVERBEE_ADMIN__PASSWORD=changeme

volumes:
  serverbee-data:
```

Then start the server:

```bash
docker compose up -d
```

The dashboard is now available at `http://your-server-ip:9527`.

## Option 2: Install Script

Download and run the install script for a quick binary installation:

```bash
curl -fsSL https://raw.githubusercontent.com/ZingerLittleBee/ServerBee/main/deploy/install.sh | sudo bash -s -- server
```

This will download the appropriate binary for your platform, create a systemd service, and start ServerBee automatically.

After installation, manage your ServerBee instance with the `serverbee` CLI:

```bash
sudo serverbee status              # View status
sudo serverbee upgrade -y           # Upgrade to latest
sudo serverbee restart              # Restart services
sudo serverbee config               # View config
sudo serverbee config set <key> <value>  # Update config
sudo serverbee uninstall server --purge   # Uninstall + remove data
```

## First Login

1. Open your browser and navigate to `http://your-server-ip:9527`
2. Log in with the default credentials:
   - **Username:** `admin`
   - **Password:** `admin` (or the password you configured)
3. **Change the default password immediately** via Settings > Security

<Callout type="warn">
If you are using the default password, the server will display a prominent warning in the logs at startup. Always change the default password before exposing the server to the internet.
</Callout>

## Adding Your First Agent

### Step 1: Generate a one-time enrollment code

Sign in to the web UI as an admin, go to **Settings**, and generate a one-time
**enrollment code**, then copy it. The code is **single-use** and **short-lived**
(default 10 minute expiry); mint a fresh one each time you onboard a new agent.

### Step 2: Configure and start the agent

If you onboarded the agent via the install script (Option 2), the configuration file is generated automatically and you can skip this manual step.

On the VPS you want to monitor, create a minimal agent configuration file that only points at the server:

```toml title="/etc/serverbee/agent.toml"
server_url = "http://your-server-ip:9527"
```

For collector, logging, and other tunable options, see [Agent Configuration](/en/docs/agent) and the [full configuration reference](/en/docs/configuration).

On its first start, register the agent with the one-time enrollment code:

```bash
SERVERBEE_ENROLLMENT_CODE=YOUR_ONE_TIME_CODE serverbee-agent
```

The agent will:

1. Contact the server and present the one-time enrollment code
2. Register itself and receive a per-server authentication token (the code is consumed on this first successful registration)
3. Save the token to the config file for future connections (the enrollment code is no longer needed)
4. Begin reporting system metrics immediately

You should see the new server appear in your dashboard within a few seconds.

<Callout type="info">
When the agent can read a stable machine identifier, repeated auto-registration from the same host reuses the existing server entry instead of creating duplicate placeholders.
</Callout>
