ServerBee 提供多种安装方式，你可以根据自身环境和需求选择最合适的方案。

## 前置条件

根据安装方式的不同，你需要准备以下环境之一：

- **Docker Compose**（推荐）：Docker 20.10+ 和 Docker Compose V2
- **安装脚本**：Linux (amd64/arm64) 系统，支持 systemd

## 方式一：Docker Compose（推荐）

这是最快的安装方式，适合大多数用户。

创建 `docker-compose.yml` 文件：

```yaml title="docker-compose.yml"
services:
  serverbee-server:
    image: ghcr.io/zingerlittlebee/serverbee-server:latest
    container_name: serverbee-server
    ports:
      - "9527:9527"
    volumes:
      - serverbee-data:/data
    environment:
      - SERVERBEE_ADMIN__USERNAME=admin
      - SERVERBEE_ADMIN__PASSWORD=your_secure_password
    restart: unless-stopped

volumes:
  serverbee-data:
```

启动服务：

```bash
docker compose up -d
```

<Callout type="warn">
务必将 `SERVERBEE_ADMIN__PASSWORD` 修改为你自己的强密码。如果不设置密码，系统会自动生成随机密码并打印到日志中。
</Callout>

启动完成后，打开浏览器访问 `http://your-server-ip:9527` 即可进入管理面板。

## 方式二：安装脚本

适用于 Linux 服务器，一键完成安装、配置和 systemd 服务注册。

**安装 Server：**

```bash
curl -fsSL https://raw.githubusercontent.com/ZingerLittleBee/ServerBee/main/deploy/install.sh | sudo bash -s -- server
```

安装脚本会自动完成以下操作：

1. 检测系统架构（amd64/arm64）
2. 从 GitHub Releases 下载对应的二进制文件
3. 安装到 `/usr/local/bin/`
4. 创建 `/etc/serverbee/` 配置目录并生成默认配置
5. 创建 systemd service unit
6. 启动服务并设置开机自启

**安装 Agent：**

在需要监控的每台服务器上执行：

```bash
curl -fsSL https://raw.githubusercontent.com/ZingerLittleBee/ServerBee/main/deploy/install.sh | sudo bash -s -- agent \
  --server-url http://your-server-ip:9527 \
  --enrollment-code YOUR_ONE_TIME_CODE
```

**管理命令：**

安装完成后，使用 `serverbee` CLI 管理你的实例：

```bash
sudo serverbee status              # 查看所有组件状态
sudo serverbee upgrade -y           # 升级到最新版
sudo serverbee restart              # 重启所有服务
sudo serverbee config               # 查看当前配置
sudo serverbee config set <key> <value>  # 修改配置
sudo serverbee uninstall agent      # 卸载 Agent
sudo serverbee uninstall server --purge   # 卸载服务端并清除数据
```

## 首次登录

1. 打开浏览器访问 `http://your-server-ip:9527`
2. 使用默认账户 `admin` 和你设置的密码登录
3. 如果未设置密码，查看 Server 启动日志获取自动生成的密码

<Callout type="warn">
首次登录后请立即修改默认密码。进入「设置」页面即可修改。
</Callout>

## 添加第一台 Agent

完成 Server 安装后，按以下步骤添加监控节点：

### 第一步：生成一次性注册码

以管理员身份登录管理面板，进入「设置」页面，生成一个一次性注册码（Enrollment Code）并复制。

注册码为**单次使用**且**短时有效**（默认 10 分钟过期），仅在 Agent 首次注册时使用。每需要接入一台新 Agent，就在设置页重新生成一个新码。

### 第二步：安装并启动 Agent

在被监控的服务器上执行一键安装脚本，通过参数传入 Server 地址和一次性注册码：

```bash
curl -fsSL https://raw.githubusercontent.com/ZingerLittleBee/ServerBee/main/deploy/install.sh | sudo bash -s -- agent \
  --server-url http://your-server-ip:9527 \
  --enrollment-code YOUR_ONE_TIME_CODE
```

脚本会自动检测平台架构、下载对应二进制、写入配置、注册 systemd 服务并启动 Agent。

更多采集、日志等可调选项请参考 [Agent 配置](/cn/docs/agent)和[完整配置参考](/cn/docs/configuration)。

首次启动时，Agent 会自动向 Server 注册：注册码在首次注册成功时即被消费，Agent 获取每台服务器专属的 Token 并写回配置文件。后续启动改用持久化的 Token，无需再提供注册码。如果某个 Agent 丢失了 Token 需要重新接入，请在设置页重新生成一个新的注册码。

<Callout type="info">
当 Agent 能读取稳定的机器标识时，重复自动注册会复用已有服务器记录，而不是继续生成新的占位条目。
</Callout>

### 第三步：验证连接

回到管理面板的仪表盘页面，你应该能看到新添加的服务器已上线，实时指标开始更新。
