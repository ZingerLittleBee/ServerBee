ServerBee 使用 [figment](https://github.com/SergioBenitez/Figment) 库加载配置，支持 TOML 配置文件和环境变量两种方式。

## 配置加载优先级

配置项的加载顺序如下（后者覆盖前者）：

1. **内置默认值** -- 程序中硬编码的默认配置
2. **TOML 配置文件** -- `/etc/serverbee/server.toml` 或 `/etc/serverbee/agent.toml`
3. **环境变量** -- 以 `SB_` 为前缀的环境变量

即环境变量具有最高优先级，可以在不修改配置文件的情况下覆盖任意配置项。

## 环境变量映射规则

环境变量遵循以下命名规则：

- 统一使用 `SB_` 前缀
- 配置层级用 `__`（双下划线）分隔
- 全部大写

**映射示例：**

| TOML 配置 | 环境变量 |
|-----------|----------|
| `server.listen` | `SB_SERVER__LISTEN` |
| `server.data_dir` | `SB_SERVER__DATA_DIR` |
| `database.path` | `SB_DATABASE__PATH` |
| `database.max_connections` | `SB_DATABASE__MAX_CONNECTIONS` |
| `auth.session_ttl` | `SB_AUTH__SESSION_TTL` |
| `auth.auto_discovery_key` | `SB_AUTH__AUTO_DISCOVERY_KEY` |
| `admin.username` | `SB_ADMIN__USERNAME` |
| `admin.password` | `SB_ADMIN__PASSWORD` |
| `retention.records_days` | `SB_RETENTION__RECORDS_DAYS` |
| `geoip.enabled` | `SB_GEOIP__ENABLED` |
| `geoip.mmdb_path` | `SB_GEOIP__MMDB_PATH` |
| `log.level` | `SB_LOG__LEVEL` |
| `log.file` | `SB_LOG__FILE` |

## 完整 server.toml 参考

```toml title="/etc/serverbee/server.toml"
# ============================================================
# ServerBee Server 配置文件
# ============================================================

# --- 基础配置 ---
[server]
# 监听地址和端口
# 默认: "0.0.0.0:9527"
listen = "0.0.0.0:9527"

# 数据存储目录，数据库文件和其他持久化数据存放于此
# 默认: "/var/lib/serverbee"
data_dir = "/var/lib/serverbee"

# --- 数据库配置 ---
[database]
# SQLite 数据库文件名（相对于 data_dir）
# 默认: "serverbee.db"
path = "serverbee.db"

# 连接池最大连接数
# SQLite 写锁是全局的，多连接主要用于并发读
# 默认: 10
max_connections = 10

# --- 认证配置 ---
[auth]
# Session 过期时间（秒），采用滑动过期策略
# 每次有效请求自动延长过期时间
# 默认: 86400 (24 小时)
session_ttl = 86400

# Agent 自动注册密钥
# 留空则 Server 首次启动时自动生成并打印到日志
# 默认: "" (自动生成)
auto_discovery_key = ""

# --- 初始管理员 ---
[admin]
# 初始管理员用户名
# 仅在 users 表为空时生效
# 默认: "admin"
username = "admin"

# 初始管理员密码
# 留空则自动生成随机密码并打印到日志
# 默认: "" (自动生成)
password = ""

# --- 速率限制 ---
[rate_limit]
# 登录接口速率限制：每 IP 每分钟最大请求数
# 默认: 5
login_max = 5

# Agent 注册接口速率限制：每 IP 每分钟最大请求数
# 默认: 3
register_max = 3

# --- 数据保留策略 ---
[retention]
# 分钟级指标记录保留天数
# 默认: 7
records_days = 7

# 小时级聚合指标保留天数
# 默认: 90
records_hourly_days = 90

# GPU 指标记录保留天数
# 默认: 7
gpu_records_days = 7

# Ping 探测记录保留天数
# 默认: 7
ping_records_days = 7

# 审计日志保留天数
# 默认: 180
audit_logs_days = 180

# --- GeoIP 地理位置 ---
[geoip]
# 是否启用 IP 地理位置查询
# 默认: false
enabled = false

# MaxMind MMDB 数据库文件路径
# 默认: ""
mmdb_path = ""

# --- 日志配置 ---
[log]
# 日志级别: trace / debug / info / warn / error
# 默认: "info"
level = "info"

# 日志文件路径，留空则输出到 stdout
# 默认: ""
file = ""

# --- OAuth 配置（可选）---
# [oauth.github]
# enabled = true
# client_id = ""
# client_secret = ""

# [oauth.google]
# enabled = true
# client_id = ""
# client_secret = ""

# [oauth.oidc]
# enabled = true
# issuer_url = ""
# client_id = ""
# client_secret = ""
```

## 完整 agent.toml 参考

```toml title="/etc/serverbee/agent.toml"
# ============================================================
# ServerBee Agent 配置文件
# ============================================================

# Server 的 WebSocket 地址（必填）
# 使用 ws:// 或 wss://（反向代理 HTTPS 时）
server_url = "ws://your-server-ip:9527"

# Agent Token，注册成功后自动写入
# 首次运行时留空，通过 Discovery Key 注册后自动填入
token = ""

# --- 采集配置 ---
[collector]
# 指标采集和上报间隔（秒）
# 此值可被 Server 端 Welcome 消息中的 report_interval 覆盖
# 默认: 3
interval = 3

# 是否启用 GPU 指标采集
# 需要编译时启用 gpu feature 且服务器安装 NVIDIA 驱动
# 默认: false
enable_gpu = false

# 是否启用温度采集
# 默认: true
enable_temperature = true

# --- 日志配置 ---
[log]
# 日志级别: trace / debug / info / warn / error
# 默认: "info"
level = "info"

# 日志文件路径，留空则输出到 stdout
# 默认: ""
file = "/var/log/serverbee-agent.log"
```

## 重要默认值汇总

下表列出了所有关键的默认值，便于快速参考：

### Server 默认值

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 监听端口 | `9527` | HTTP 和 WebSocket 共用端口 |
| 数据目录 | `/var/lib/serverbee` | 数据库和持久化数据 |
| 数据库文件 | `serverbee.db` | SQLite 数据库 |
| 连接池大小 | `10` | 最大并发数据库连接 |
| Session 有效期 | `86400` 秒（24 小时） | 滑动过期 |
| 管理员用户名 | `admin` | 仅首次初始化时使用 |
| 登录速率限制 | `5` 次/分钟/IP | 防暴力破解 |
| 注册速率限制 | `3` 次/分钟/IP | 防滥用注册 |
| 分钟级指标保留 | `7` 天 | 自动清理过期数据 |
| 小时级指标保留 | `90` 天 | 长期趋势分析 |
| GPU 指标保留 | `7` 天 | 与分钟级指标一致 |
| Ping 记录保留 | `7` 天 | 与分钟级指标一致 |
| 审计日志保留 | `180` 天 | 半年审计记录 |
| GeoIP | 关闭 | 需手动配置 MMDB 文件 |
| 日志级别 | `info` | 推荐生产环境使用 |

### Agent 默认值

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 采集间隔 | `3` 秒 | 可被 Server 动态调整 |
| GPU 采集 | 关闭 | 需要 NVIDIA 驱动 |
| 温度采集 | 开启 | 默认采集传感器温度 |
| 日志级别 | `info` | 推荐生产环境使用 |

### 内部默认值（不可配置）

以下是系统内部的固定参数，供了解系统行为参考：

| 参数 | 值 | 说明 |
|------|-----|------|
| Agent 上报间隔 | 3 秒 | 由 Server Welcome 消息控制 |
| 心跳间隔 | 30 秒 | Server 向 Agent 发送 Ping |
| 离线判定 | 30 秒 | 无上报则判定离线 |
| 离线检测扫描 | 10 秒 | 后台扫描 Agent 连接状态 |
| 指标写入 | 每 1 分钟 | 从内存缓存批量写入数据库 |
| 小时聚合 | 每 1 小时 | 分钟级数据聚合为小时级 |
| WebSocket JSON 帧上限 | 1 MB | 单条 JSON 消息最大体积 |
| WebSocket Binary 帧上限 | 64 KB | 终端数据帧最大体积 |
| 命令执行超时 | 300 秒 | 远程命令最大执行时间 |
| 命令输出上限 | 512 KB | 超出部分截断 |
| 命令长度上限 | 8 KB | 单条命令最大长度 |
| 并发命令上限 | 5 | 每台 Agent 最多并发执行 |
| 终端会话上限 | 3 | 每台服务器最多并发终端 |
| 终端空闲超时 | 10 分钟 | 无输入自动断开 |
| 告警采样窗口 | 10 分钟 | 最近 10 个采样点 |
| 告警触发比例 | 70% | 超过 70% 采样超阈值才触发 |
| 告警通知去抖 | 5 分钟 | 同一规则+服务器最短通知间隔 |
| 重连退避上限 | 30 秒 | 指数退避最大等待时间 |
| 重连抖动 | +/-20% | 避免雷群效应 |
